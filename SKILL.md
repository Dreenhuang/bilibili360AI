---
name: bili2doc
description: 当用户提到B站视频转文字、批量导出B站视频文字稿、360AI识别视频内容时使用。触发关键词：「B站文章」「360识别」「360AI识别」「B站转文字」「导出B站视频」「B站文字稿」「视频转docx」「360AI浏览器识别」「B站批量导出」。基于360AI浏览器的全文AI分析功能，自动将B站视频内容转录为.docx文档。支持批量处理、夜间无人值守模式、断点续传、下载清单管理。
---

[任务]
    **单视频模式**：用户提供单个B站视频URL → 打开360AI浏览器，触发全文AI分析，导出为.docx文件。

    **批量模式**：用户提供B站博主主页URL或UID → 提取全部视频列表，逐个处理，批量导出所有视频的文字稿。

    **夜间模式**：默认开启。处理过程中浏览器自动最小化，不打扰用户工作。需要操作时自动恢复窗口。

[依赖检测]
    Skill 启动时第一步自动执行。

    必需：
    - 360AI浏览器（安装路径：D:\360AI\360aibrowser\Application\360aibrowser.exe）→ 缺失则提示用户安装
    - Python 3.8+ → 缺失则提示安装
    - pyautogui、pywin32、pillow → 缺失则自动执行 pip install
    - video_list.json → 批量模式必需，缺失则先运行提取脚本
    - batch_state.json → 断点续传用，首次运行自动生成

    可选：
    - bilibili-api-python → 有则可用API提取视频列表，无则提示手动提供
    - logs/ 目录 → 有则查看历史运行记录

    安装策略：
    - 360AI浏览器缺失 → 提示用户从 https://ai.360.com/ 下载安装
    - Python依赖缺失 → Agent 自主执行 pip install pyautogui pywin32 pillow
    - video_list.json缺失（批量模式）→ 自动运行 scripts/extract_videos.py 提取

[第一性原则]
    **浏览器唯一**：所有AI分析必须通过360AI浏览器完成，不尝试其他浏览器或API替代方案。
    **坐标精准**：扩展UI不在DOM内，必须使用屏幕级坐标点击。不同分辨率需重新校准坐标。
    **频率保护**：360AI有每日调用限制（约33次/天），检测到频率限制时自动冷却，不持续轰炸API。
    **断点续传**：每次处理后保存状态到batch_state.json，中断后可从断点继续，不重复处理。
    **夜间友好**：默认最小化浏览器运行，不打扰用户工作。需要操作时自动恢复窗口。
    **联网优先**：涉及360AI更新、bilibili-api兼容性时先WebSearch确认再动手。

[工作流程]
    [第一步：识别用户意图]
        根据用户输入判断模式：
        - 单个视频URL（含BV号）→ 单视频模式
        - 博主主页URL或UID → 批量模式
        - 查看进度/状态 → 运行view_manifest.py展示可视化清单
        - 挑选视频 → 运行view_manifest.py --select进入交互选择

    [第二步：批量模式 - 展示清单并等待用户选择]
        用户提供博主URL或确认批量处理后：
        1. 运行 scripts/view_manifest.py 展示完整视频清单
           表格格式：序号 | 状态 | BV号 | 视频标题 | 时长 | 播放量
           状态标识：✅完成 📥已下载 ❌失败 ⏳待处理
        2. 输出统计摘要：总视频数、已完成、失败、待处理、进度百分比
        3. 等待用户选择：
           - 用户输入序号：如 "5" 或 "1,3,5" 或 "1-10" 或 "1-5,10,15-20"
           - 用户输入 "全部"：处理所有待处理视频
           - AI主动询问："请输入要处理的视频序号，或输入'全部'处理所有待处理视频"
        4. 确认选择后，运行 scripts/view_manifest.py --select 保存选中列表
        5. 生成 selected_videos.json

    [第三步：环境检查]
        自动检查：
        - 360AI浏览器是否安装
        - Python依赖是否完整
        - 项目目录是否存在
        缺失则引导安装或自动安装

    [第三步：执行处理]
        单视频模式：
        1. 启动/重启360AI浏览器
        2. 导航到视频页面（地址栏点击→Ctrl+A→Ctrl+V→Enter）
        3. 验证导航成功（URL包含bilibili.com/video/）
        4. 点击「AI模式」按钮（坐标：1193, 86）
        5. 点击「全文」按钮（坐标：1070, 95）
        6. 等待AI分析（120秒，夜间模式期间最小化浏览器）
        7. PIL图像分析检测AI结果（侧边栏亮度<100=频率限制）
        8. 如AI成功→点击「导出」按钮（坐标：1733, 163）
        9. 回车确认下载，监控下载目录
        10. 检查.docx文件是否成功下载

        批量模式：
        1. 先运行 extract_videos.py 提取视频列表（如video_list.json不存在）
        2. 运行 view_manifest.py 展示可视化清单
        3. 等待用户选择视频（交互式挑选）
        4. 保存选中列表到 selected_videos.json
        5. 运行 bili2doc.py --select 处理选中视频
        6. 每个视频执行单视频模式流程
        7. 频率限制时自动冷却（30分钟→2小时指数退避）
        8. 每处理15个视频重启浏览器
        9. 视频间延迟10分钟
        10. 完成后重新运行view_manifest.py核对清单

    [第四步：处理完成核对]
        处理完成后必须执行：
        1. 重新运行 scripts/view_manifest.py 生成最新可视化清单
        2. 核对：
           - 本次选中的视频是否都已处理
           - 是否有漏处理的视频
           - 是否有新增的失败视频
           - 状态更新是否正确
        3. 输出完整核对报告：
           - 本次新增成功数
           - 累计完成数
           - 仍待处理数
           - 失败列表
           - 进度百分比

    [第五步：结果输出]
        单视频模式：
        - 成功：告知用户文件路径和大小
        - 失败（频率限制）：告知冷却时间，建议稍后重试
        - 失败（其他）：告知错误原因和截图位置

        批量模式：
        - 输出统计：成功数、失败数、总耗时
        - 输出失败视频列表（如有）
        - 告知报告文件位置（night_run_report.json）
        - 夜间模式：告知浏览器已最小化

[输出风格]
    **语态**：
    - 像自动化运维汇报：准确、简洁、有数据
    - 完成了就报告成功，失败就说明原因和下一步

    **原则**：
    - × 绝不隐瞒失败原因
    - × 绝不在频率限制期间承诺"马上就好"
    - × 绝不跳过环境检查直接执行
    - ✓ 每次处理输出明确的视频编号、BV号、标题
    - ✓ 失败时输出具体原因和解决建议
    - ✓ 批量处理输出进度百分比和预计剩余时间

    **典型表达**：
    - "✅ #3 BV1Eu4y1L7UN《芯片简史》导出成功，文件：xxx.docx (45KB)"
    - "⚠️ 频率限制触发，已自动冷却60分钟。今日配额约33次，建议明日00:00后重试。"
    - "批量处理完成：成功32个，失败57个（均为频率限制）。报告：night_run_report.json"

[频率限制策略]
    360AI/Doubao API有每日调用次数限制：
    - 触发特征：侧边栏显示"操作过于频繁，请稍后再试"
    - 检测方式：PIL图像分析，侧边栏亮度<100判定为错误页面
    - 冷却策略：指数退避 30→60→90→120分钟
    - 根本解决：每日配额约33次，00:00重置，超限只能等待
    - 优化建议：增加视频间隔时间，减少触发频率

[坐标配置]
    基于1920x1080分辨率，用户实测确认：
    ```
    全文 = (1070, 95)      # 点击触发全文AI分析
    AI模式 = (1193, 86)    # 切换到AI模式
    导出 = (1733, 163)     # 导出分析结果
    地址栏 = (400, 55)     # 地址栏区域（用于导航）
    ```
    不同分辨率需使用坐标测量工具重新校准。
