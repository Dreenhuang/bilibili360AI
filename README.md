# bili2doc - B站视频AI转文字工具

[![Python](https://img.shields.io/badge/Python-3.8%2B-blue.svg)](https://www.python.org/)
[![License](https://img.shields.io/badge/License-MIT-green.svg)](LICENSE)
[![Platform](https://img.shields.io/badge/Platform-Windows-lightgrey.svg)](https://www.microsoft.com/windows)

使用 **360AI浏览器** 的「全文」AI分析功能，将B站视频内容自动转录为 `.docx` 文档。

## ✨ 特性

- 🎯 **精准定位** - 基于屏幕级坐标点击，绕过DOM限制直接操作扩展UI
- 🔄 **批量处理** - 自动遍历博主全部视频，批量导出文字稿
- 💾 **断点续传** - 状态持久化，中断后可从断点继续
- 🛡️ **频率保护** - 智能检测360AI频率限制，自动冷却恢复
- 📊 **进度可视** - 实时日志输出，截图记录关键节点
- 🚀 **一键启动** - 双击批处理文件即可运行全套流程
- 🌙 **夜间模式** - 自动最小化浏览器，不打扰日常工作（默认开启）

## 📋 系统要求

| 项目 | 要求 |
|------|------|
| **操作系统** | Windows 10/11 |
| **Python版本** | 3.8+ |
| **浏览器** | 360AI浏览器 1.1+ |
| **安装路径** | `D:\360AI\360aibrowser\Application\360aibrowser.exe` |

## 🔧 快速开始

### 1. 安装依赖

```bash
pip install pyautogui pywin32 pillow
```

### 2. 提取视频列表

```bash
python scripts/extract_videos.py --uid 315846984
```

### 3. 开始批量导出

```bash
python bili2doc.py              # 夜间模式运行（默认，不打扰工作）
python bili2doc.py --day-mode   # 白天模式运行（浏览器保持前台）
python bili2doc.py --count 5    # 只处理5个（测试用）
```

### 4. 夜间自动运行（推荐）

**方式一：手动启动**
双击 `night_run.bat` 文件，程序会在后台运行，浏览器自动最小化。

**方式二：定时启动**
运行 `create_scheduled_task.bat`，设置每天自动运行的时间（如23:00）。
程序会在指定时间自动启动，完成所有视频处理后自动最小化浏览器。

### 5. 一键启动（传统模式）

双击运行项目根目录下的 `启动bili2doc.bat` 文件。

## ⚙️ 配置说明

所有配置在 `bili2doc.py` 的 `Config` 类中：

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `WAIT_AI_SECONDS` | 120 | AI分析等待时间（秒） |
| `BATCH_SIZE` | 5 | 每批处理视频数 |
| `INTER_VIDEO_DELAY` | 600 | 视频间隔（秒） |
| `RATE_LIMIT_BASE_COOLDOWN` | 1800 | 基础冷却时间（30分钟） |
| `AI_ERROR_BRIGHTNESS` | 100 | 错误页面亮度阈值 |
| `DOWNLOAD_DIR` | `D:\360AI浏览器\下载` | 下载目录 |

## ⚠️ 注意事项

1. **坐标适配** - 扩展UI坐标基于1920x1080分辨率，不同分辨率需重新校准
2. **频率限制** - 360AI有每日调用次数限制（约33次/天），超限后需等待重置
3. **浏览器状态** - 运行前请确保360AI浏览器已安装且扩展已启用
4. **窗口焦点** - 运行时请勿操作电脑，避免窗口焦点被切换
5. **下载目录** - 请确保下载目录有足够磁盘空间

## 🛠️ 故障排除

### Q: 下载的文件为空
A: 检查AI分析是否成功，侧边栏是否显示"操作过于频繁"

### Q: 导航到错误页面
A: 确认地址栏坐标配置正确（默认400, 55）

### Q: 浏览器无法置顶
A: 运行前关闭其他全屏应用，确保浏览器可正常获取焦点

### Q: 频率限制一直不解除
A: 可能是每日配额已耗尽，需等待次日重置（约00:00）
