## 一、项目简介

bili2doc 是一款自动化工具，使用 **360AI浏览器** 的「全文」AI分析功能，将B站视频内容自动转录为 `.docx` 文字文档。

**核心功能：**
- 自动打开B站视频页面
- 点击360AI扩展的「全文」功能
- 等待AI分析生成视频文字稿
- 自动导出并下载为 `.docx` 文件
- 批量处理整个博主的所有视频

## 四、详细操作流程

### 4.1 单个视频处理流程

```
1. 打开360AI浏览器
   ↓
2. 导航到视频页面
   (点击地址栏 → Ctrl+A → Ctrl+V → Enter)
   ↓
3. 点击「AI模式」按钮
   (坐标: 1193, 86)
   ↓
4. 点击「全文」按钮
   (坐标: 1070, 95)
   ↓
5. 等待AI分析 (120秒)
   ↓
6. 检测分析结果
   (PIL亮度分析: <100=失败, >180=成功)
   ↓
7. 点击「导出」按钮
   (坐标: 1733, 163)
   ↓
8. 回车确认下载
   ↓
9. 监控下载目录
   (检测新生成的.docx文件)
   ↓
10. 保存处理状态
```

### 4.2 批量处理流程

```
1. 读取 video_list.json
   ↓
2. 加载 batch_state.json (断点续传)
   ↓
3. 过滤已完成的视频
   ↓
4. 按批次处理 (每批5个)
   ↓
5. 每处理完一批：
   - 更新状态文件
   - 输出进度报告
   ↓
6. 全部完成后生成最终报告
```

## 五、配置说明

### 5.1 核心配置项

所有配置在 `bili2doc.py` 的 `Config` 类中：

| 配置项 | 默认值 | 说明 | 调整建议 |
|--------|--------|------|---------|
| `WAIT_AI_SECONDS` | 120 | AI分析等待时间 | 视频较长可增加 |
| `BATCH_SIZE` | 5 | 每批处理数量 | 根据电脑性能调整 |
| `INTER_VIDEO_DELAY` | 600 | 视频间隔(秒) | 频率限制时增加 |
| `RATE_LIMIT_BASE_COOLDOWN` | 1800 | 基础冷却(秒) | 频率限制严重时增加 |

### 5.2 坐标配置

```python
COORDS = {
    "全文":   (1070, 95),    # 「全文」按钮
    "AI模式": (1193, 86),    # 「AI模式」按钮
    "导出":   (1733, 163),   # 「导出」按钮
    "地址栏": (400, 55),     # 地址栏区域
}
```

**⚠️ 坐标适配说明：**
- 基于1920x1080分辨率
- 不同分辨率需重新测量
- 使用 `mouse_coord.py` 工具获取坐标

## 六、常见问题

### Q1: 提示"操作过于频繁"怎么办？

**原因：** 360AI有每日调用次数限制（约33次/天）

**解决方案：**
1. 等待冷却（默认30分钟）
2. 等待次日重置（00:00）
3. 更换360AI账号

### Q2: 下载的文件为空？

**原因：** AI分析失败，侧边栏没有内容

**解决方案：**
1. 检查是否触发频率限制
2. 检查网络连接
3. 增加AI等待时间 (`WAIT_AI_SECONDS`)

### Q3: 导航到错误页面？

**原因：** 地址栏坐标不正确

**解决方案：**
1. 确认浏览器窗口最大化
2. 重新测量地址栏坐标
3. 使用 `mouse_coord.py` 工具校准

### Q4: 浏览器无法置顶？

**原因：** 其他应用抢占焦点

**解决方案：**
1. 运行前关闭其他全屏应用
2. 不要操作电脑，让程序自动运行
3. 确保360AI浏览器窗口可以正常最大化

### Q5: 如何断点续传？

**说明：** 程序自动保存状态到 `batch_state.json`

**操作：**
1. 直接重新运行 `python bili2doc.py`
2. 程序会自动跳过已完成的视频
3. 从上次中断处继续

## 七、高级用法

### 7.1 自定义下载目录

修改 `bili2doc.py` 中的 `DOWNLOAD_DIR`：

```python
DOWNLOAD_DIR = r"D:\我的文档\bili2doc输出"
```

### 7.2 修改AI等待时间

对于长视频，可能需要更长的AI分析时间：

```python
WAIT_AI_SECONDS = 180  # 3分钟
```

### 7.3 添加更多频率限制保护

增加视频间隔时间和冷却时间：

```python
INTER_VIDEO_DELAY = 900          # 15分钟间隔
RATE_LIMIT_BASE_COOLDOWN = 3600  # 1小时基础冷却
```

## 九、性能优化建议

### 9.1 加快处理速度

1. **减少视频间隔**（可能导致频率限制）
2. **减少AI等待时间**（可能导致分析不完整）
3. **增加批次大小**（可能导致浏览器不稳定）

### 9.2 提高成功率

1. **增加视频间隔**（减少频率限制触发）
2. **增加冷却时间**（给API恢复时间）
3. **定期重启浏览器**（每15个视频）
